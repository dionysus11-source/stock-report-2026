# 제안서: 실제 데이터를 활용한 기업 분석 개선

## 목표
`stock-fundamental` 스킬에서 사용하는 모의 데이터를 네이버 증권 등에서 수집한 **실시간 재무 데이터**로 대체하여 분석 결과의 실효성을 확보합니다.

## 배경
현재 `analyze.py`는 하드코딩된 "Best" 점수(16점)만 반환합니다. 실제로 유용한 도구가 되려면 사용자의 체크리스트(PER, PBR, ROE 등)를 실제 지표와 비교하여 판단해야 합니다.

## 설계
- **데이터 출처**: 토스 증권 (Toss Securities) 웹사이트 (`tossinvest.com`).
- **방법**: `stock-capture`와 유사하게 **Headless Browser (Puppeteer/Playwright)**를 사용하여 페이지를 로드하고 재무 데이터를 추출합니다. (SPA 구조 대응)
- **전처리**: 검색 시 종목 코드 앞에 **'A'를 붙여서 조회**합니다 (예: `005930` -> `A005930`).
- **매핑**: 토스 증권 화면에 표시되는 재무 지표를 체크리스트 항목과 매핑합니다.

## 위험 요소
- **구현 복잡도**: 단순 HTML 파싱보다 Headless Browser 사용이 더 무겁고 복잡합니다.
- **데이터 가용성**: 토스 증권 UI에서 제공하는 정보가 네이버 증권보다 적을 수 있습니다. (제공되지 않는 지표는 '확인 불가' 처리)
